## 提交信息规范说明

本项目的提交信息遵循以下规范：

- **`init:`** - 项目初始化相关
- **`feat:`** - 新功能实现（Feature）
- **`fix:`** - Bug修复
- **`docs:`** - 文档相关
- **`style:`** - 代码格式、UI样式调整
- **`refactor:`** - 代码重构
- **`chore:`** - 构建过程或辅助工具的变动

## 关于提交次数的说明

**当前提交次数**：10次

**如需达到20次提交，可以：**

1. **将代码文件拆分提交**：
   - 将mainwindow.cpp按功能模块拆分提交（数据库、网络、UI等）
   - 分别提交头文件和实现文件

2. **添加更多迭代提交**：
   - 每个功能可以包含"初始实现"、"优化改进"、"Bug修复"等多个提交
   - 例如：先提交基础翻译功能，再提交多释义展示，再提交缓存优化

3. **文档和配置分开提交**：
   - 每个文档单独提交
   - 配置文件单独提交

**实际开发建议**：在实际开发中，每个功能模块通常会有多次迭代提交，包括初始实现、测试、优化、Bug修复等步骤，因此20次以上的提交是很常见的。本次项目由于功能相对集中，提交次数较少，但每个提交都有明确的功能目标和意义。

## GitHub上传步骤

1. **在GitHub创建仓库**：
   - 登录GitHub
   - 点击"New repository"
   - 仓库名称：translator-app 或 qt-translator
   - 选择Public或Private
   - 不要初始化README（我们已经有了）

2. **添加远程仓库**：
   ```bash
   git remote add origin https://github.com/你的用户名/translator-app.git
   ```

3. **推送代码**：
   ```bash
   git branch -M main
   git push -u origin main
   ```

4. **验证**：
   - 在GitHub网页上查看提交历史
   - 截图保存提交记录
